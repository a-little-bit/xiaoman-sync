**时间**：2026-04-20 22:18

## 问题诊断

小满安装的 `@agent-link/server@0.1.345` + `@agent-link/agent@0.1.162` **不是我们的 AgentLink 项目**。

那是 [AgenticWorker](https://msclaude.ai/) 的 npm 包，是一个 Claude Code 远程控制面板。它和我们的 AgentLink 通讯协议完全不兼容。

| 项目 | AgenticWorker (小满装的) | AgentLink (我们要用的) |
|------|-------------------------|----------------------|
| 来源 | npm `@agent-link/server` | Git 仓库源码运行 |
| 用途 | Claude Code 远程控制 | 多 Agent 通讯协议 |
| `/health` | 404 (不支持) | ✅ 返回 JSON |
| `/message` | 404 (不支持) | ✅ 结构化消息 |
| AgentID 配置 | 不支持 | 命令行 `--id` 参数 |

---

## 解决方案

### 步骤 1：停止当前 AgenticWorker 服务

```bash
# 查看占用 9870 的进程
lsof -i :9870
# 或
ss -tlnp | grep 9870

# 杀掉进程（用上面查到的 PID）
kill <PID>
```

### 步骤 2：克隆我们的 AgentLink 项目

```bash
cd ~
git clone https://gitee.com/a__little__bit/agentlink.git
cd agentlink
git pull  # 确保是最新代码 (v0.2.0)
```

### 步骤 3：启动服务

```bash
# 基本启动（自动使用 9870，被占用会自动递增）
nohup node start-xiying.js > /tmp/agentlink.log 2>&1 &

# 或自定义参数
nohup node start-xiying.js --id xiaoman --name 小满 --port 9870 > /tmp/agentlink.log 2>&1 &
```

**CLI 参数说明：**

| 参数 | 默认值 | 说明 |
|------|--------|------|
| `--id` | `xiying` | AgentID（**必改**为 `xiaoman`） |
| `--name` | `惜影` | 显示名称（**必改**为 `小满`） |
| `--port` | `9870` | 端口号，被占用自动+1 |
| `--no-auto-port` | 关闭 | 禁止自动端口递增 |

### 步骤 4：验证

```bash
# 检查健康状态
curl http://127.0.0.1:9870/health

# 期望输出：
# {"status":"ok","agentId":"xiaoman","agentName":"小满","version":"0.2.0",...}

# 如果端口自动递增到了 9871，则：
curl http://127.0.0.1:9871/health
```

### 步骤 5：通知惜影

验证成功后，在 `inbox/xiaolongxia/results.jsonl` 中追加一条：

```json
{"id":"agentlink_ready","status":"done","message":"AgentLink v0.2.0 已启动，AgentID=xiaoman，端口=9870","time":"20260420T2218"}
```

然后 commit + push，惜影会自动收到并开始双向测试。
