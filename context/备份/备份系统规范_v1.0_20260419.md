## 三、Git版本控制

### 3.1 提交规范

```
[类型] 简短描述 {日期}

详细说明（可选）
```

**类型**：
- `[日志]` - 对话记录相关
- `[备份]` - 核心文件备份
- `[配置]` - 配置变更
- `[项目]` - 项目进展
- `[修复]` - 问题修复

### 3.2 回滚操作

```bash
# 查看提交历史
git log --oneline -20

# 回滚到指定版本（保留工作区修改）
git reset --soft <commit_hash>

# 回滚到指定版本（丢弃所有修改）
git reset --hard <commit_hash>

# 查看某个文件的历史版本
git log --oneline -- ./MEMORY.md

# 恢复某个文件到指定版本
git checkout <commit_hash> -- ./MEMORY.md
```

### 3.3 远程仓库地址

- **GitHub**: https://github.com/a-little-bit/xiaoman-sync
- **Gitee**: https://gitee.com/a__little__bit/xiaoman-sync

## 五、恢复流程

### 5.1 从Git恢复

```bash
# 1. 克隆仓库
git clone https://github.com/a-little-bit/xiaoman-sync.git

# 2. 查看历史
git log --oneline

# 3. 恢复到指定日期
git checkout <commit_hash>
```

### 5.2 从本地备份恢复

```bash
# 查看可用备份
ls -la ./备份/

# 恢复某个备份
cp -r ./备份/core_backup_20260419/* ./
```

### 5.3 从飞书恢复

1. 登录飞书
2. 进入「小满档案」知识库
3. 下载对应文件

## 八、监控与告警

### 8.1 备份成功检查

每次备份后检查：
- Git提交是否成功
- 双端推送是否成功
- 备份文件是否完整

- 推送失败：重试3次，仍失败则通知用户
