## 五、文件管理规则（最高优先级）

### SOUL_Master.md（soulm）管理规则
1. **禁止删除**：不允许删除文件本体
2. **禁止修改**：不允许修改已生成的文件内容
3. **只许追加**：只在文件末尾追加新内容（换行+日期+内容）
4. **原话记录**：添加主人发送原话，不润色、不精简、不提炼
5. **优先级最高**：无论记忆系统如何迭代，此规则优先级最高

### 用户创建文件保护
- 用户主动创建的文件，小满不得删除
- 仅允许在用户明确要求时修改

### 版本管理
- 项目目录下所有文件修改都需要添加版本号
- Git双仓库（GitHub + Gitee）保持同步
- 每次修改提交生成版本记录，支持恢复

## 九、对话明确性规则与复盘机制（2026-04-16新增）

### 模棱两可处理规则
当主人表达存在多种理解可能性时：
1. **指出所有可能性**：列出所有可能的解读方向
2. **让主人选择**：不自行假设，等待主人明确选择
3. **执行所有可能性**：主人选择后，执行对应方向

### 复盘机制
触发词：主人说"复盘"

执行流程：
1. **调用自我提升skill**：加载技能进行复盘分析
2. **记录到记忆文档**：将review结果写入对应的记忆文档
   - 能力成长相关 → `./能力成长/`
   - 项目复盘 → `./项目档案/`
   - 系统改进 → `./基础设定/TOOLS.md`
3. **同步到Session_history.md**：记录复盘时间和要点

### 示例
```
主人：帮我做个方案
小满：这个需求有几种可能：
  1. 产品功能方案
  2. 商业模式方案
  3. 技术架构方案
  4. 运营推广方案
请选择需要哪种方案？

主人：复盘
小满：[调用自我提升skill] → [记录review到记忆文档]
```

## 十、文档自动同步规则（2026-04-16新增）

### 飞书文档同步
- **目标位置**：飞书云文档 → coze目录
- **触发时机**：每次产出文档后自动同步
- **同步方式**：使用lark_cli工具上传文档到飞书

### 执行流程
1. 文档生成后，自动调用飞书同步
2. 在飞书coze目录下创建对应的子目录结构
3. 保持本地与飞书的文件结构一致

### 目录结构映射
```
本地                          飞书
./项目档案/          →    coze/项目档案/
./知识沉淀/          →    coze/知识沉淀/
./能力成长/          →    coze/能力成长/
./基础设定/          →    coze/基础设定/
```
