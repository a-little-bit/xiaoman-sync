# 小满日志系统 v3.0

> 2026-04-19 升级：强化原始对话记录机制，确保数据完整性

---

## 一、核心原则

### 1.1 双轨记录原则
- **原始对话**：完整保存用户消息和小满回复，不做任何加工
- **整理内容**：提取关键信息，分类归档，便于检索和分析

### 1.2 实时记录原则
- 每次对话结束时，立即追加写入当日日志
- 不依赖定时任务，定时任务仅用于汇总和同步

## 三、原始对话记录格式

### 3.1 对话原文格式

```markdown
# 对话原文 {YYYY-MM-DD}

## 会话信息
- SessionKey: {session_key}
- Channel: {channel}
- 开始时间: {start_time}
- 最后更新: {last_update}

---

## 对话记录

### [{HH:MM:SS}] 用户消息
{用户原始消息完整内容}

### [{HH:MM:SS}] 小满回复
{小满回复完整内容}

### [{HH:MM:SS}] 工具调用
```
工具: {tool_name}
目的: {purpose}
结果: {success/fail}
摘要: {result_summary}
```

---
（每次对话追加以上格式）
```

### 3.2 工具调用记录格式（JSONL）

每行一条记录，便于追加：
```json
{"timestamp":"2026-04-19T18:20:00+08:00","tool":"bash","purpose":"检查目录结构","success":true,"duration_ms":150}
{"timestamp":"2026-04-19T18:20:05+08:00","tool":"read_file","purpose":"读取配置文件","success":true,"duration_ms":50}
```

## 四、记录机制

### 4.1 实时记录流程

**每次对话结束时执行**：

1. **检查当日日志文件是否存在**
   - 不存在则创建
   - 存在则追加

2. **写入原始对话**
   - 用户消息（完整原文）
   - 小满回复（完整内容）
   - 工具调用记录（如有）

3. **提取结构化信息**
   - 识别对话类型
   - 写入对应分类日志

### 4.2 对话类型识别

| 关键词 | 类型 | 写入目录 |
|--------|------|----------|
| 记住、定义、设定、配置 | 配置变更 | 结构化日志/配置变更/ |
| 学习、能力、技能、成长 | 能力成长 | 结构化日志/能力成长/ |
| 项目、夜市、独立站 | 项目进展 | 结构化日志/项目进展/ |
| @技能名、使用技能 | 技能使用 | 结构化日志/技能使用/ |
| 问题、错误、失败 | 问题解决 | 结构化日志/问题解决/ |
| 想法、灵感、点子 | 灵感捕捉 | 结构化日志/灵感捕捉/ |

### 4.3 定时任务处理

**每日 22:00 - 日志同步任务**：
1. 检查当日是否有原始对话记录
2. 如有，生成每日总结
3. 执行 Git 同步（提交到 GitHub + Gitee）

**每周日 21:00 - 周复盘任务**：
1. 汇总本周所有对话
2. 生成每周复盘
3. 更新能力雷达
4. 同步到 Git

**每月最后一天 23:00 - 月报告任务**：
1. 生成月度成长报告
2. 清理过期临时文件
3. 归档历史日志

## 五、快速记录命令

### 5.1 追加原始对话

```bash
# 追加到当日对话原文
echo "
### [$(date +%H:%M:%S)] 用户消息
{用户消息}

### [$(date +%H:%M:%S)] 小满回复
{回复内容}
---" >> ./日志/原始记录/对话原文_$(date +%Y%m%d).md
```

### 5.2 追加工具调用

```bash
# 追加工具调用记录
echo '{"timestamp":"$(date -Iseconds)","tool":"{tool}","purpose":"{purpose}","success":{true/false}}' >> ./日志/原始记录/工具调用_$(date +%Y%m%d).jsonl
```

## 八、历史数据迁移

### 8.1 缺失日志补录

当前缺失：2026-04-12 至 2026-04-19（8天）

1. 如有记忆痕迹（MEMORY.md、SOUL.md更新记录），提取关键事件
2. 创建当日对话原文文件，标注"系统补录"
