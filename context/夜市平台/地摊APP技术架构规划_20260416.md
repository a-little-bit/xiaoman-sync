## 一、项目概述

### 1.1 项目定位

**夜市平台**是一款面向移动地摊经济的O2O服务平台，旨在为地摊商户提供数字化经营工具，为消费者提供便捷的本地生活服务。核心功能包括：

- 商户入驻与管理（摊位信息、菜品上架、营业时间）
- 用户端服务（浏览、下单、支付、评价）
- 配送调度（骑手管理、实时定位、路径规划）
- 智能运营（客流分析、热度地图、经营报表）

### 1.2 核心需求指标

| 指标项 | 目标值 | 说明 |
|--------|--------|------|
| 用户规模 | 亿级 | 支持高并发、高可用架构 |
| 多端覆盖 | 全平台 | 微信/支付宝/抖音小程序 + Web + PC + App |
| 系统可用性 | 99.95%+ | 年故障时间 < 4.38小时 |
| 接口响应时间 | P99 < 200ms | 核心链路性能保障 |
| 数据存储 | PB级 | 支持未来5年业务增长 |

### 1.3 差异化定位

与美团外卖等成熟平台相比，夜市平台聚焦以下差异化价值：

1. **轻量化运营**：极简入驻流程，3分钟完成摊位上线
2. **场景适配**：专为流动摊位设计的移动端交互
3. **低成本启动**：适合小微商户的SaaS化服务
4. **政策友好**：支持政府规范化管理需求

| 维度 | UniApp | Taro | Flutter | React Native |

3. **一键多端**：同一套代码覆盖小程序、H5、App
4. **成本优势**：中小团队首选，降低60%开发工作量
5. **案例背书**：美团、京东、阿里等大厂在使用

## 五、数据库设计

### 5.1 核心表结构

```sql
-- 用户表
CREATE TABLE `users` (
  `id` BIGINT PRIMARY KEY AUTO_INCREMENT,
  `phone` VARCHAR(20) NOT NULL UNIQUE COMMENT '手机号',
  `nickname` VARCHAR(50) COMMENT '昵称',
  `avatar` VARCHAR(255) COMMENT '头像URL',
  `balance` DECIMAL(10,2) DEFAULT 0 COMMENT '余额',
  `status` TINYINT DEFAULT 1 COMMENT '状态:1正常 2禁用',
  `created_at` DATETIME DEFAULT CURRENT_TIMESTAMP,
  `updated_at` DATETIME DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
  INDEX idx_phone(phone)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4;

-- 商户/摊位表
CREATE TABLE `merchants` (
  `id` BIGINT PRIMARY KEY AUTO_INCREMENT,
  `user_id` BIGINT NOT NULL COMMENT '商户用户ID',
  `name` VARCHAR(100) NOT NULL COMMENT '摊位名称',
  `logo` VARCHAR(255) COMMENT 'logo',
  `category_id` INT COMMENT '分类ID',
  `description` TEXT COMMENT '描述',
  `address` VARCHAR(255) COMMENT '地址',
  `latitude` DECIMAL(10,8) COMMENT '纬度',
  `longitude` DECIMAL(11,8) COMMENT '经度',
  `business_hours` JSON COMMENT '营业时间',
  `status` TINYINT DEFAULT 0 COMMENT '状态:0待审核 1正常 2禁用',
  `rating` DECIMAL(2,1) DEFAULT 5.0 COMMENT '评分',
  `monthly_sales` INT DEFAULT 0 COMMENT '月销量',
  `created_at` DATETIME DEFAULT CURRENT_TIMESTAMP,
  INDEX idx_user_id(user_id),
  INDEX idx_status(status),
  INDEX idx_location(latitude, longitude)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4;

-- 商品表
CREATE TABLE `products` (
  `id` BIGINT PRIMARY KEY AUTO_INCREMENT,
  `merchant_id` BIGINT NOT NULL,
  `category_id` INT COMMENT '分类ID',
  `name` VARCHAR(100) NOT NULL COMMENT '商品名',
  `cover_image` VARCHAR(255) COMMENT '封面图',
  `images` JSON COMMENT '图片列表',
  `price` DECIMAL(10,2) NOT NULL COMMENT '价格',
  `original_price` DECIMAL(10,2) COMMENT '原价',
  `stock` INT DEFAULT 0 COMMENT '库存',
  `description` TEXT COMMENT '描述',
  `specs` JSON COMMENT '规格',
  `status` TINYINT DEFAULT 1 COMMENT '状态:1上架 2下架',
  `sales` INT DEFAULT 0 COMMENT '销量',
  `created_at` DATETIME DEFAULT CURRENT_TIMESTAMP,
  INDEX idx_merchant_id(merchant_id),
  INDEX idx_status(status)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4;

-- 订单表
CREATE TABLE `orders` (
  `id` BIGINT PRIMARY KEY AUTO_INCREMENT,
  `order_no` VARCHAR(32) NOT NULL UNIQUE COMMENT '订单号',
  `user_id` BIGINT NOT NULL,
  `merchant_id` BIGINT NOT NULL,
  `rider_id` BIGINT COMMENT '骑手ID',
  `status` TINYINT NOT NULL DEFAULT 0 COMMENT '状态:0待支付 1待接单 2制作中 3待取餐 4配送中 5已完成 6已取消',
  `total_amount` DECIMAL(10,2) NOT NULL COMMENT '商品总价',
  `discount_amount` DECIMAL(10,2) DEFAULT 0 COMMENT '优惠金额',
  `delivery_fee` DECIMAL(10,2) DEFAULT 0 COMMENT '配送费',
  `pay_amount` DECIMAL(10,2) NOT NULL COMMENT '实付金额',
  `pay_type` TINYINT COMMENT '支付方式:1微信 2支付宝',
  `pay_time` DATETIME COMMENT '支付时间',
  `delivery_address` VARCHAR(255) COMMENT '配送地址',
  `delivery_latitude` DECIMAL(10,8) COMMENT '配送纬度',
  `delivery_longitude` DECIMAL(11,8) COMMENT '配送经度',
  `remark` VARCHAR(255) COMMENT '备注',
  `created_at` DATETIME DEFAULT CURRENT_TIMESTAMP,
  `updated_at` DATETIME DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
  INDEX idx_order_no(order_no),
  INDEX idx_user_id(user_id),
  INDEX idx_merchant_id(merchant_id),
  INDEX idx_status(status),
  INDEX idx_created_at(created_at)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4;

-- 订单明细表
CREATE TABLE `order_items` (
  `id` BIGINT PRIMARY KEY AUTO_INCREMENT,
  `order_id` BIGINT NOT NULL,
  `product_id` BIGINT NOT NULL,
  `product_name` VARCHAR(100) NOT NULL COMMENT '商品快照',
  `cover_image` VARCHAR(255) COMMENT '图片快照',
  `price` DECIMAL(10,2) NOT NULL COMMENT '单价快照',
  `quantity` INT NOT NULL DEFAULT 1,
  `specs` JSON COMMENT '规格快照',
  `subtotal` DECIMAL(10,2) NOT NULL COMMENT '小计'

-- 骑手表
CREATE TABLE `riders` (
  `id` BIGINT PRIMARY KEY AUTO_INCREMENT,
  `user_id` BIGINT NOT NULL,
  `name` VARCHAR(50) NOT NULL,
  `phone` VARCHAR(20) NOT NULL,
  `status` TINYINT DEFAULT 0 COMMENT '状态:0离线 1在线 2配送中',
  `latitude` DECIMAL(10,8) COMMENT '当前位置纬度',
  `longitude` DECIMAL(11,8) COMMENT '当前位置经度',
  `total_orders` INT DEFAULT 0 COMMENT '累计配送',
  `rating` DECIMAL(2,1) DEFAULT 5.0,
  `created_at` DATETIME DEFAULT CURRENT_TIMESTAMP
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4;

-- 评价表
CREATE TABLE `reviews` (
  `id` BIGINT PRIMARY KEY AUTO_INCREMENT,
  `order_id` BIGINT NOT NULL,
  `user_id` BIGINT NOT NULL,
  `merchant_id` BIGINT NOT NULL,
  `rating` TINYINT NOT NULL COMMENT '评分1-5',
  `content` TEXT COMMENT '评价内容',
  `images` JSON COMMENT '图片',
  `reply` TEXT COMMENT '商家回复',
  `created_at` DATETIME DEFAULT CURRENT_TIMESTAMP
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4;
```

## 八、风险评估与应对

### 8.1 技术风险

| 风险 | 等级 | 应对措施 |
|------|------|----------|
| 高并发超出预期 | 高 | 提前压测、限流熔断、弹性扩容 |
| 支付安全问题 | 极高 | 使用官方SDK、签名验证、回调校验 |
| 数据丢失 | 高 | 定期备份、多副本存储 |
| 服务雪崩 | 中 | 熔断降级、超时控制 |

### 8.2 业务风险

| 风险 | 等级 | 应对措施 |
|------|------|----------|
| 用户增长缓慢 | 中 | 运营补贴、地推获客 |
| 商户入驻率低 | 中 | 地推团队、重点城市突破 |
| 竞争对手复制 | 中 | 差异化运营、深耕区域 |
| 政策合规风险 | 中 | 法务审核、资质齐全 |

### 8.3 运营风险

| 风险 | 等级 | 应对措施 |
|------|------|----------|
| 骑手配送效率低 | 中 | 调度算法优化、激励政策 |
| 恶意订单/刷单 | 中 | 风控系统、设备指纹 |
| 用户差评影响 | 低 | 客服体系、主动回访 |
