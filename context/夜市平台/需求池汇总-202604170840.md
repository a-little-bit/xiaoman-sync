## 三、新增需求详细设计

### 3.1 积分体系设计

#### 3.1.1 积分定位

积分体系是用户增长和留存的核心驱动力，通过"积分获取→积分消耗→积分活动"闭环，实现用户行为的正向激励。

#### 3.1.2 积分获取场景

| 行为类型 | 积分获取 | 每日上限 | 说明 |
|---------|---------|---------|------|
| **消费返积分** | 消费1元 = 1积分 | 无上限 | 按实际支付金额计算 |
| **每日签到** | 第1天5积分，第7天30积分 | 1次/日 | 连续签到阶梯奖励 |
| **首次完成行为** | +20积分 | 仅首次 | 首单、首评、首次分享等 |
| **评价返积分** | 发布图文评价 +10积分 | 3次/日 | 被点赞额外+2积分/个 |
| **分享返积分** | 分享到微信/朋友圈 +5积分 | 2次/日 | 需要有曝光才算 |
| **任务完成** | 按任务难度 5-50积分 | 按任务 | 核查任务、日常任务等 |
| **绑定手机号** | +10积分 | 仅首次 | 完善用户画像 |
| **完善个人资料** | +5积分 | 仅首次 | 昵称/头像/口味偏好 |

#### 3.1.3 积分消耗场景

| 消耗类型 | 所需积分 | 说明 |
|---------|---------|------|
| **兑换优惠券** | 100积分 = 1元 | 100积分起兑，上限500积分/券 |
| **兑换礼品** | 按商品定价 | 积分商城实物/虚拟商品 |
| **积分抽奖** | 10-50积分/次 | 实物奖品、优惠券、积分翻倍券 |
| **积分抵现** | 100积分 = 1元 | 下单时可选是否使用 |
| **兑换会员特权** | 500积分/月 | 免广告/专享券等 |
| **捐赠公益** | 100积分起 | 参与平台公益活动 |

#### 3.1.4 积分规则设计

```
┌─────────────────────────────────────────────────────────────────┐
│                        积分规则设计                              │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│  【基础规则】                                                    │
│  1. 积分有效期：获得后365天内有效，过期清零                       │
│  2. 积分不可转让、不可合并账号                                    │
│  3. 积分不找零、不兑换现金                                        │
│  4. 退款订单积分扣除，已使用积分需返还                             │
│                                                                 │
│  【等级体系】 ← 建议配合会员体系                                  │
│  ├── Lv1 吃瓜群众（0-999积分）：基础权益                          │
│  ├── Lv2 夜市常客（1000-4999积分）：额外9.5折                    │
│  ├── Lv3 烟火达人（5000-19999积分）：额外9折+专属客服             │
│  └── Lv4 夜市老炮（20000+积分）：额外8.5折+优先配送+线下活动邀请  │
│                                                                 │
│  【安全规则】                                                    │
│  1. 刷积分行为检测：同一IP/设备/账号频繁操作                      │
│  2. 积分异常冻结：核实前积分暂时冻结                               │
│  3. 举报奖励：发现刷积分奖励举报者额外积分                        │
└─────────────────────────────────────────────────────────────────┘
```

#### 3.1.5 积分活动类型

| 活动类型 | 规则描述 | 频率 | 效果 |
|---------|---------|------|------|
| **限时双倍积分** | 指定时间段消费得双倍积分 | 每周五18:00-22:00 | 提升夜市高峰消费 |
| **新店开业冲量** | 指定摊位开业期间3倍积分 | 按需配置 | 帮助新摊位冷启动 |
| **节日主题活动** | 节假日推出专属积分加倍 | 国庆/春节等 | 提升节日活跃 |
| **积分抽奖转盘** | 消耗积分抽奖，实物/积分/优惠券 | 每日1次 | 消耗积分、提升趣味 |
| **积分任务赛** | 周榜排名前10额外奖励积分 | 每周结算 | 激励用户持续活跃 |
| **邀请积分加倍** | 被邀请人首单邀请人得3倍积分 | 仅首次 | 提升裂变效果 |

#### 3.1.6 积分体系技术要点

- **实时性**：积分变动需实时更新展示
- **一致性**：积分扣减需幂等，防止重复扣减
- **可追溯**：每笔积分变动需记录来源/去向
- **对账**：每日凌晨生成积分对账单

### 3.2 游客核查任务系统

#### 3.2.1 系统定位

核查任务是平台获取摊位信息真实性的重要手段，通过激励用户完成核查任务，收集并验证摊位信息，确保平台数据的准确性和时效性。

#### 3.2.2 任务类型

| 任务类型 | 任务描述 | 核查内容 | 奖励 | 难度 |
|---------|---------|---------|------|------|
| **新摊位发现** | 发现平台未收录的新摊位 | 摊位名称、位置、品类、照片 | 20积分 + 3元现金 | ★★☆ |
| **营业状态验证** | 核实摊位是否正常营业 | 营业状态、营业时间、排队长短 | 5积分 + 0.5元现金 | ★☆☆ |
| **摊位信息核实** | 核实已有摊位信息是否准确 | 菜品、价格、环境照片 | 10积分 + 1元现金 | ★★☆ |
| **菜品上新记录** | 记录摊位新推出的菜品 | 菜品名、价格、图片、口味 | 15积分 + 2元现金 | ★★☆ |
| **食品安全巡查** | 拍摄摊位卫生情况 | 证照公示、环境卫生、食品存放 | 30积分 + 5元现金 | ★★★ |
| **服务态度评价** | 收集用户对摊位的服务反馈 | 服务评分、具体反馈文字 | 5积分 | ★☆☆ |
| **排队时间记录** | 记录当前摊位排队时长 | 排队人数、预估等待时间 | 3积分/次 | ★☆☆ |

#### 3.2.3 核查任务流程

```
┌─────────────────────────────────────────────────────────────────┐
│                      核查任务执行流程                            │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│  步骤1：任务领取                                                 │
│  ├── 用户打开"核查中心"Tab                                      │
│  ├── 系统推送附近可接任务（500m内）                              │
│  ├── 用户点击"立即领取"锁定任务（15分钟内有效）                  │
│  └── 锁定后其他用户不可重复领取                                  │
│                                                                 │
│  步骤2：现场核查                                                 │
│  ├── 用户到达摊位位置（需开启定位校验）                          │
│  ├── 系统自动记录：定位+时间戳+设备信息                          │
│  ├── 按要求拍摄照片/填写信息                                     │
│  └── 完成后点击"提交核查"                                       │
│                                                                 │
│  步骤3：审核验证                                                 │
│  ├── 系统初步校验：照片质量+定位偏差+时间有效性                   │
│  ├── 交叉验证：同一摊位需2-3人独立核查                           │
│  ├── 人工抽检：AI识别异常时进入人工复核                           │
│  └── 审核通过后积分/现金奖励发放                                 │
│                                                                 │
│  步骤4：数据应用                                                 │
│  ├── 核查结果更新摊位信息（待确认状态）                          │
│  ├── 连续3次验证一致 → 信息标记"已核实"                         │
│  ├── 信息应用于：实时状态更新、新摊位发现奖励摊主                │
│  └── 核查记录计入用户贡献值                                      │
└─────────────────────────────────────────────────────────────────┘
```

#### 3.2.4 核查奖励机制

| 奖励类型 | 计算方式 | 发放时间 | 说明 |
|---------|---------|---------|------|
| **积分奖励** | 按任务类型固定奖励 | 审核通过后即时 | 实时可见，增强感知 |
| **现金奖励** | 按任务类型固定金额 | 次日到账 | 需绑定支付宝/微信 |
| **新手保护期** | 前10次任务额外+50%奖励 | 前10次 | 降低门槛，培养习惯 |
| **连续核查奖励** | 连续7天核查 +50积分 | 第7天结算 | 培养用户粘性 |
| **月度核查达人** | 月度核查数量Top10额外奖励 | 月末结算 | 实物奖品/荣誉称号 |
| **差错率奖励** | 核查准确率>95%额外奖励 | 季度结算 | 质量激励 |

#### 3.2.5 防作弊机制

| 防护维度 | 具体措施 | 触发条件 | 处理方式 |
|---------|---------|---------|---------|
| **定位校验** | 必须到达摊位附近才能提交 | 距离>50m | 提交被拒，扣除10积分 |
| **时间校验** | 任务领取后需在15分钟内完成 | 超时 | 任务自动释放，需重新领取 |
| **设备指纹** | 同一设备频繁提交触发风控 | 1小时内>3次 | 进入人工审核 |
| **IP校验** | 同一IP多账号提交 | 同一IP>2账号 | 关联账号全部冻结 |
| **照片校验** | AI识别照片是否真实拍摄 | 照片重复/PS痕迹 | 审核不通过 |
| **行为分析** | 异常用户行为模式检测 | 异常模式匹配 | 降低任务推送频率 |
| **交叉验证** | 多用户独立核查同一摊位 | 3人核查结果不一致 | 进入人工复核 |
| **举报机制** | 用户可举报虚假核查 | 举报核实 | 举报者+50积分，被举报者-100积分 |

### 3.3 集市广场功能

#### 3.3.1 功能定位

集市广场是平台提供的"摊位集合"服务，允许符合条件的商家创建和管理自己的主题集市，为用户提供更丰富、更具特色的消费场景。

#### 3.3.2 集市定义与类型

| 集市类型 | 描述 | 示例 | 创建门槛 |
|---------|------|------|---------|
| **主题集市** | 围绕特定主题聚合摊位 | 烧烤一条街、甜品集中营 | 中 |
| **时段集市** | 特定时段营业的摊位集合 | 深夜食堂、早茶一条街 | 中 |
| **地域集市** | 地理区域内自然形成的集市 | XX夜市、XX广场美食街 | 低 |
| **品牌集市** | 同一品牌/连锁摊位集合 | 某某小厨连锁、网红品牌区 | 高 |
| **节庆集市** | 节日/活动期间临时集市 | 跨年美食节、啤酒节 | 低 |

#### 3.3.3 创建门槛设计

| 门槛维度 | 具体要求 | 建议数值 | 说明 |
|---------|---------|---------|------|
| **摊位数量** | 集市内摊位数量 | ≥5个 | 确保集市丰富度 |
| **营业时长** | 每月最低营业天数 | ≥15天 | 避免空壳集市 |
| **平均评分** | 所有摊位平均评价分 | ≥4.0分 | 保障集市质量 |
| **平台入驻时长** | 摊主在平台入驻时间 | ≥30天 | 有一定运营记录 |
| **订单量** | 近30天累计订单 | ≥100单 | 验证摊主运营能力 |
| **保证金** | 集市创建需缴纳 | ¥500-2000 | 约束行为，保证服务 |
| **资质证明** | 场地方需提供 | 营业执照/租赁合同 | 合规性要求 |
| **管理员配置** | 集市需指定管理员 | 1人 | 负责日常管理 |

| **违规处理** | 食品安全/虚假宣传：立即清退，保证金不退 |
| **升级规则** | 连续3月考核优秀，可申请升级门槛要求 |
