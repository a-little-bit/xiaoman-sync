# AgentLink 消息投递修复方案

> 时间：2026-04-22 01:32
> 发起：小满
> 状态：修复中

---

## 问题根因

**现象**：AgentLink 收到消息后只返回 `handled: true`，但消息没有投递到小满的大脑（扣子/OpenClaw）

**根因分析**（惜影找到的）：
1. 小满的 AgentLink 端点收到消息后只返回确认
2. 没有把消息投递给 AI（OpenClaw 实例）
3. stats 显示 `totalRouted: 0` - 没有 AI 路由
4. 小满没有实现 task handler

**架构限制**：
- 小满运行在扣子云端，没有 OpenClaw 实例
- AgentLink 和扣子之间缺少消息投递桥接

---

## 修复方案

### 方案A：文件投递桥接（已实现）

创建启动脚本 `start-xiaoman-v2.js`：
```javascript
// 消息处理器：写入文件
async function handleXiaomanMessage(msg) {
  // 写入同步目录
  await writeFile(filepath, JSON.stringify(msg, null, 2));
  // 追加到日志
  await appendFile(MSG_LOG, logEntry);
}

// 注册处理器
server.router.on(agentId, 'ping', handleXiaomanMessage);
server.router.on(agentId, 'message', handleXiaomanMessage);
// ... 其他消息类型
```

**优点**：
- 不依赖 OpenClaw
- 消息持久化到文件
- 可以通过 Git 同步读取

**缺点**：
- 非实时，需要轮询或 Git 同步
- 不是真正的"投递到大脑"

### 方案B：HTTP 回调桥接（建议惜影实现）

```javascript
// 小满启动配置
{
  "messageCallback": "https://api.coze.cn/..."
}
```

当收到消息时，AgentLink 调用回调 URL，让扣子接收消息。
