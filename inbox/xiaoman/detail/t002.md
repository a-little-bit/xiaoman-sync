## 一、任务目标

创建一套完整的Linux定时同步脚本，实现小满与小虾的自动化协作：
1. 定时拉取任务
2. 检测新任务
3. 执行任务（人工介入时通知）
4. 推送结果

---

## 二、核心需求

### 2.1 主同步脚本 (sync_daemon.sh)

**功能要求**：
- 每5分钟执行一次同步检查
- 拉取GitHub和Gitee双端更新
- 检测 `inbox/xiaoman/tasks.jsonl` 是否有新任务
- 有新任务时：
  - 更新 `state/status.json` 的 `working_on` 字段
  - 发送桌面通知或写入通知文件
- 无新任务时静默退出

**同步策略**：
```bash
# 优先从GitHub拉取，失败则从Gitee拉取
git pull github main --rebase || git pull gitee main --rebase
# 双端推送
git push github main && git push gitee main
```

### 2.2 任务检测脚本 (check_tasks.sh)

**功能要求**：
- 读取 `inbox/xiaoman/tasks.jsonl`
- 对比 `inbox/xiaolongxia/.tasks_read` 标记文件
- 输出新任务列表（JSON格式）
- 更新标记文件

**输出格式**：
```json
{
  "count": 2,
  "tasks": [
    {"id":"t001","title":"奔走夜市需求深度调研","priority":1},
    {"id":"t002","title":"创建Linux定时同步脚本","priority":1}
  ]
}
```

### 2.3 任务完成脚本 (complete_task.sh)

**功能要求**：
- 参数：任务ID、状态、结果摘要、产出文件列表
- 追加结果到 `inbox/xiaolongxia/results.jsonl`
- 追加原始记录到 `originals/results.jsonl`
- 更新 `state/status.json`
- 提交并推送

**使用示例**：
```bash
./scripts/complete_task.sh t001 done "完成需求调研" "outputs/xiaolongxia/2026-04-19/t001_research/"
```

### 2.4 通知脚本 (notify.sh)

**功能要求**：
- 支持多种通知方式：
  - 桌面通知（notify-send）
  - 写入通知文件 `inbox/xiaolongxia/notifications.txt`
  - 可选：发送到指定HTTP接口
- 通知内容包含：任务ID、标题、优先级、截止时间

## 三、定时任务配置

### 3.1 Systemd Service（推荐）

创建 `xiaoman-sync.service` 和 `xiaoman-sync.timer`：
- 每5分钟执行一次
- 自动重启失败的服务
- 日志输出到 journald

### 3.2 Crontab（备选）

提供crontab配置示例：
```cron
*/5 * * * * /workspace/projects/xiaoman-sync/scripts/sync_daemon.sh >> /var/log/xiaoman-sync.log 2>&1
```
